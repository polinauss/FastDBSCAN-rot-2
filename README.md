# PointCloud Classifier (ROS 2)

Кластеризация и классификация объектов в облаках точек (PCD) с использованием FastDBSCAN и ROS 2.
Проект предназначен для автоматического выделения и классификации дорожных объектов: деревья, автомобили, велосипедисты/пешеходы, а также неизвестные объекты.

## Возможности

- Загрузка облаков точек из PCD-файлов или из ROS-топика /input_pointcloud
- Фильтрация шумов (статистическое удаление выбросов) и удаление земли (z < 0.2 м)
- Кластеризация методом FastDBSCAN
- Вычисление свойств кластеров: центроид, габариты (ширина, высота, глубина), плотность
- Классификация по геометрическим признакам:
  * TREE – высота > 2.2 м, ширина и глубина < 1.5 от высоты
  * CAR – высота 0.8-2.5 м, длина 3.2-7.5 м, ширина 1.2-3.2 м
  * CYCLIST – высота 1.2-1.9 м, длина 1.2-2.2 м, ширина 0.5-1.2 м
  * UNKNOWN – остальное
- Публикация результатов в ROS-топик /cluster_results
- Сохранение результатов в CSV-файл с разделителем ;

## Входные данные

- Формат: PCD (текстовый или бинарный), точки типа x y z
- Источник: лазерное сканирование дорожной сцены (лидар)
## Выходные данные

### ROS-топик /cluster_results

Сообщение pointcloud_classifier/msg/ClusterResultArray содержит массив структур. Каждый элемент имеет поля:

- id (int32) – номер кластера
- type_name (string) – TREE / CAR / CYCLIST / UNKNOWN
- type_code (int32) – 0=UNKNOWN, 1=TREE, 2=CAR, 3=CYCLIST
- center_x, center_y, center_z (float32) – координаты центроида
- num_points (int32) – количество точек в кластере
- width, height, depth (float32) – размеры по X, Z, Y
- density (float32) – плотность точек (точек/м^3)

### Файл classification_results.txt (CSV)

Разделитель – точка с запятой. Пример строки:

ID;Тип;Центр_X;Центр_Y;Центр_Z;Точек;Ширина;Высота;Глубина;Плотность
0;TREE;1.23;4.56;7.89;123;0.89;2.50;0.76;12.3

## Установка и сборка

Требования: Ubuntu 22.04 / 24.04, ROS 2, PCL, colcon, git, cmake.
'''''''''''''''''''''''''''''''''''
Установите зависимости:

sudo apt update
sudo apt install ros-jazzy-pcl-ros ros-jazzy-pcl-conversions pcl-tools

Клонируйте и соберите пакет:

cd ~/ros2_ws/src
git clone https://github.com/your-username/pointcloud_classifier.git
cd ~/ros2_ws
colcon build --packages-select pointcloud_classifier
source install/setup.bash

## Запуск

### 1. Запуск узла классификатора

ros2 run pointcloud_classifier classifier_node

Можно менять параметры через --ros-args -p:

- fastdbscan_K (по умолчанию 80) – число начальных кластеров для K‑means этапа
- fastdbscan_t (0.12) – доля точек для выборки (12%)
- fastdbscan_eps (0.5) – радиус кластеризации в метрах
- fastdbscan_minPts (40) – минимальное число точек в кластере
- filter_mean_k (50) – параметр статистического удаления выбросов
- filter_stddev_mult (1.0) – порог выбросов
- output_file (classification_results.txt) – имя выходного CSV-файла
- save_to_file (true) – сохранять результаты в файл

Пример с изменением параметров:

ros2 run pointcloud_classifier classifier_node \
  --ros-args -p fastdbscan_K:=100 -p fastdbscan_t:=0.15 -p output_file:=~/my_results.txt''''

### 2. Публикация PCD-файла
В другом терминале выполните:

ros2 run pointcloud_classifier pcd_publisher \
  --ros-args -p pcd_file:=/путь/к/файлу.pcd -p publish_rate_hz:=0.5

- publish_rate_hz – частота публикации (0.5 = раз в 2 секунды).
  Для однократной отправки используйте 0.01 и прервите после первого сообщения (Ctrl+C).

### 3. Просмотр результатов в реальном времени

ros2 topic echo /cluster_results

## Примечания

- FastDBSCAN является стохастическим (использует случайную инициализацию). При каждом запуске результаты для одного и того же файла могут незначительно отличаться. Для детерминированных результатов зафиксируйте seed в коде или используйте обычный DBSCAN (метод performClustering).
- Обработка одного облака из 8000 точек занимает 100-150 мс.
- Поддерживаются только точки типа pcl::PointXYZ (без цвета, без интенсивности).
